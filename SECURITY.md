# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| 0.1.x | Yes |

## Reporting a Vulnerability

Report vulnerabilities privately to the maintainers before opening a public issue.

When reporting:

- Describe the impact and affected workflow.
- Include reproduction steps and sample paths when relevant.
- Avoid attaching sensitive ShareFile content unless explicitly requested through a secure channel.
